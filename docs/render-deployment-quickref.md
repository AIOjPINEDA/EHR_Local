# Render Deployment Quick Reference

## Backend URLs (replace with actual URLs after deployment)
- Backend API: https://consultamed-backend.onrender.com
- Health Check: https://consultamed-backend.onrender.com/health
- API Documentation: https://consultamed-backend.onrender.com/docs

## Verification Commands
```bash
# Check health endpoint
curl -I https://consultamed-backend.onrender.com/health

# Check API docs
curl -I https://consultamed-backend.onrender.com/docs

# Test authentication (get token)
curl -X POST "https://consultamed-backend.onrender.com/auth/token" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "username=test@example.com&password=testpassword"

# Test protected endpoint with token
curl -X GET "https://consultamed-backend.onrender.com/protected-endpoint" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

## Environment Variables Checklist
- [ ] DATABASE_URL updated with actual PostgreSQL connection
- [ ] DATABASE_MODE=render_cloud
- [ ] JWT_SECRET_KEY generated by Render
- [ ] ACCESS_TOKEN_EXPIRE_MINUTES=60
- [ ] FRONTEND_URL set to frontend URL
- [ ] ENVIRONMENT=production
- [ ] DEBUG=false

## Important Notes
1. The backend will deploy with a temporary DATABASE_URL
2. You MUST update this after creating the database
3. Redeploy the service after updating DATABASE_URL
4. WeasyPrint dependencies are installed automatically via render_build.sh
